# M2 review pass (2026-07-16)

8-angle recall review over the M2 diff (`ed6d041..fefbbd0`, the tier-1 AI ordering milestone),
run the same day the milestone landed. ~33 candidates, 10 survived verification. All fixed in
the follow-up commit except the deliberate holds below.

## Fixed

- **POST `/api/order-job` TOCTOU**: the in-flight guard sat before three `await`s; two
  concurrent POSTs could spawn two paid model calls racing on one overlay file. The guard and
  the `liveJob` assignment now have no `await` between them, and an in-memory `liveRecord`
  closes the sibling GET race (a just-started job briefly read as orphaned).
- **PATCH `/api/order` lost updates**: read-modify-write was unserialized; auto-apply vs
  dismiss (or two tabs) could drop a sticky flag. Now chained like review saves.
- **`?order=ai` silent tier-0 fallback**: an eval could confidently compare tier0-vs-tier0.
  Now 409s when no fresh overlay exists (CLI already errored).
- **Greedy `{...}` extraction**: first-`{`-to-last-`}` regex over the model reply, duplicated
  in job and judge. Replaced by one string-aware brace-balanced extractor in
  `server/src/claude-cli.ts`, which also owns the `claude -p` spawn (tool-less flags in one
  place); `tools/order-eval.mjs` imports it from dist. Unit-tested.
- **Indicator honesty**: `orderApplied` now derives from "the applier actually reordered", not
  from the decision alone â€” a fail-open apply can no longer render the AI label over tier-0
  order (R-026). Overlay state moved to `web/src/useOrderOverlay.ts` per the hooks convention.
- **CLI/`--ai-order`**: now writes the job record (audit trail parity with the POST path) and
  shares one compile pass with `--export` (was a full second tree-sitter pass, ~30s on very
  large ranges).
- **`applyOrderOverlay` waste**: no longer builds the full manifest (chunk summaries + a
  render-to-measure pass) just to validate; a shared `storySplit` feeds both it and
  `buildOrderManifest`, imports come from a single-pass map, and the overlay reuses the
  manifest's fingerprint instead of recomputing. `validatePermutation` takes story keys.
- **Duplication**: atomic JSON write consolidated (`json-file.ts`); `chunkLineCount` shared by
  manifest and `--dump-chunks`; `(leftovers)` sentinel now `LEFTOVERS_SECTION_ID`.
- **Conventions**: spec-number breadcrumbs dropped from comments and from the indicator
  tooltip; duplicated rationale comments trimmed to one authoritative statement each.
- `promptVersion` now rides `OrderResponse.job` (was stored but never exposed).

Re-verified after the fixes: 167 tests green; the persisted opus overlay for PR 2357 applies
byte-identically through the new path; headless UI smoke (indicator, rationale, overlay-first
section) passes; the eval tool runs against the shared dist module.

## Deliberately not fixed (with reasons)

- **Rationales threaded as a RowView prop** instead of living on the row model in `rows.ts`.
  M3 (narration) will grow row-level payloads anyway; move it then, once, with the narration
  design in hand rather than twice.
- **CLI flag precedence is position-of-check-in-file** (`--dump-graph` silently wins over
  `--ai-order`). Real but cosmetic; a proper subcommand structure is the fix and not worth it
  before the CLI surface settles.
- **`runOrderJob` retry loop shape** (one loop, two counters). Reviewed as harder to extend
  than nested retries, but it is small, correct, and unit-tested; restructure when a third
  failure class actually appears.
- **K=3/two-subjects statistical thinness** â€” recorded in spec 02's gate language and the
  baseline doc; more subjects is dogfood work, not code.
